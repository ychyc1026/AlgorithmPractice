# 队列

## 说明

## 两个.c 文件分别包含数组实现的循环队列和链表实现的队列，目前已实现核心功能：push（入队）、pop（出队）、判断空 / 满状态。学习过程中最核心、最有意思的是数组循环队列的「成环逻辑」，特整理核心要点如下，后续学习新内容会持续补充

### 一、队列的核心特性

队列是遵循「FIFO（先进先出，First In First Out）」的线性数据结构，与栈的「LIFO（后进先出）」形成鲜明对比，典型应用场景包括：
任务排队（如打印机任务、系统进程调度）；
广度优先搜索（BFS）算法；
生产者 - 消费者模型。（这些都是ai说的）

### 二、数组队列的痛点：假溢出

1. 普通数组队列的缺陷
最初的数组队列实现逻辑：
入队（push）：从队尾（rear）插入元素，rear 自增；
出队（pop）：取出队头（front）元素后，将队列后半部分整体向前移动，保证 front 始终指向数组起始位置。
问题：
时间复杂度高：pop 操作需要移动 n 个元素，时间复杂度为 O (n)；
空间浪费：频繁 pop 后，数组前半部分的空间被闲置，但新元素只能从 rear 往后插入，导致「数组物理空间未满，却提示队列已满」，这种现象称为假溢出。
2. 优化尝试：仅移动 front 指针
为解决时间复杂度问题，优化逻辑：
pop 时不再移动数组元素，直接将 front 指针自增（相当于 “丢弃” 队头元素，标记前半部分空间为闲置）；
push 仍从 rear 往后插入，直到 rear 到达数组末尾。
新问题：假溢出依然存在 ——front 前有大量闲置空间，但 rear 已到数组边界，无法继续入队。

### 三、循环队列：解决假溢出的核心方案

1. 核心思想
将数组「首尾相连」形成环形，通过模运算（%） 让 front 和 rear 指针在数组范围内 “绕圈”，复用 front 前的闲置空间，彻底解决假溢出。
2. 关键设计（0 起始数组）
（1）指针定义
front：指向队列第一个有效元素；
rear：指向队列最后一个有效元素的下一个空位（预留 1 个空位，用于区分 “空” 和 “满”）；
capacity：数组总容量（如 #define max_s 100）。
（2）核心判断逻辑

```c

// 判断队列空：front和rear指向同一位置
bool isempty(queue*q) {
    return q->front == q->rear;
}

// 判断队列满：rear的下一个位置是front
bool isfull(queue*q) {
    return (q->rear + 1) % q->capacity == q->front;
}
```

（3）成环的核心：模运算
指针移动时通过模运算实现 “绕圈”：(这是重点)

``` c
// 入队：rear指针绕圈
void push(queue*q, int a) {
    if(isfull(q)) {
        printf("满了，不能装了\n");
        return;
    }
    q->data[q->rear] = a;
    q->rear = (q->rear + 1) % q->capacity; // 核心成环操作
}

// 出队：front指针绕圈
void pop(queue*q) {
    if(isempty(q)) {
        printf("空的，不能再出了\n");
        return;
    }
    q->front = (q->front + 1) % q->capacity; // 核心成环操作
}
```

（4）辅助功能的成环适配
计算队列大小：避免 rear < front 时出现负数，公式为(q->rear - q->front + q->capacity) % q->capacity；
取队尾元素：rear 指向空位，需回退到最后一个有效元素，公式为(q->rear - 1 + q->capacity) % q->capacity。

## 四、拓展思考

若数组下标从 1 开始，成环模运算需调整为：新下标 = (当前下标 % capacity) + 1（核心逻辑不变，仅下标映射调整）；
循环队列的 “满” 状态可通过额外变量（如 size）替代 “预留空位”，但会增加逻辑复杂度；
链表队列虽无需成环，但需注意出队后重置 rear 指针（避免队尾悬空）。
